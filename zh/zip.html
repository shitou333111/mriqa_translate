零插值填充 

某些序列中有一个名为 ZIP 的选项。这是一种文件压缩技术吗？  
MRI 数据的处理通常在阵列处理器中使用快速傅里叶变换 (FFT)算法完成。这些计算要求通常需要输入数据位于一个维度为 2 的幂次方的矩阵中，例如 128x128、256x256 或 512x512。如果采集到的k空间数据未能填满此矩阵，通常会用零来填充缺失的单元，这种技术称为零填充或零插值填充 (ZIP)。 

MRI 中的 ZIP 与 *.zip 文件压缩技术无关。与邮政编码也毫无关系！
零填充通常用于在相位编码方向上扩展图像矩阵尺寸。对于 2D 成像，这种矩阵尺寸的增加是“层面内”的（例如，从 256 像素增加到 512 像素）。对于 3D 成像，ZIP 可以“跨层面”使用，以增加层块选择方向上的表观分辨率（例如，从 64 层增加到 128 层）。这种增益仅在矩阵尺寸首次翻倍时出现；此后，便无法获得更多益处。
尽管零填充并未向输入的原始数据添加任何信息，但由于减少了部分容积效应，它仍能改善图像的表观空间分辨率。零填充作为一种插值方法，对来自相邻体素的信号进行插值，使图像外观更平滑，减少“像素感”。
未使用与使用零插值填充 (ZIP) 的颅脑 MRA 示例。

右侧图像已执行 ZIP 填充，分别将矩阵尺寸增加到 256x256 和 512x512。

进阶讨论 (显示/隐藏)»

为什么k空间零填充类似于图像空间中的像素插值？其背后的数学原理有些复杂，感兴趣的读者可以参考下面列出的各种参考文献（尤其是 Bartholdi-Ernst 的论文）作为起点。

简而言之，无论选择多大的矩阵，它总是小于物体理论上无限的k空间表示，并且必须在某处被截断。截断相当于将k空间数据乘以一个“矩形”函数。这反过来又相当于在图像空间与一个 SINC 函数（矩形函数的傅里叶变换）进行卷积。零填充会在显示矩阵中产生额外的 SINC 插值像素。

一个有趣的结果是，通过零填充首次将数据点数量翻倍，实际上确实增加了超越简单插值的“新信息”。但额外的零如何能构成“新”数据呢？它们之所以有帮助，是因为普通的 FFT 无法提取 FID 中所有可用的信息。通过将点数翻倍，FFT 在提取所有“真实”数据方面变得更高效。与其说 ZIP 创造了新信息，不如说标准 FFT 在挖掘所有可用信息方面能力不足。超过首次翻倍后，便不再有优势，因为傅里叶系数现在变得完全相互关联。因此，在数据处理中通常只使用一次 ZIP 步骤。

如上所述，在 2D 成像中，ZIP 应用于“层面内”。在 3D 成像中，ZIP 可以应用于“层面内”、“跨层面”或两者兼用。不同厂商对这两种 ZIP 有不同的命名。例如，GE Healthcare 将层面内 ZIP 称为“ZIP 512”或“ZIP 1024”，而将跨层面 ZIP 称为“ZIP x 2”或“ZIP x 4”。
